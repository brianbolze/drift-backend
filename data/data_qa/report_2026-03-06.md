## Critical Issues

### C1. 70 Cartridge-Bullet Weight Mismatches (wrong bullet linkage)

**Impact:** Cartridge `bullet_weight_grains` does not match the linked bullet's `weight_grains` for 70 of 236 cartridges (30%). This means these cartridges are linked to the wrong bullet record, causing incorrect BC values to be used in ballistic calculations.

**Severity breakdown:**
- 4 severe (>50gr difference): e.g., `.350 Legend 150gr` linked to a 250gr bullet, `.300 WSM 165gr` linked to 220gr bullet
- 56 moderate (10–50gr difference): e.g., `.7mm Rem Mag 150gr` linked to 139gr CX, `.243 Win 80gr` linked to 90gr CX
- 10 slight (<10gr difference): e.g., `6mm GT 109gr` linked to 108gr ELD Match, `7.62x39 124gr` linked to 123gr FMJ

**Root cause:** The pipeline resolver is matching cartridges to bullets by product line/type rather than by weight, picking the wrong variant from multi-weight product families (e.g., linking all CX Outfitter cartridges to a single CX bullet regardless of weight).

### C2. Lapua G573 Weight Error (120gr stored as 180gr)

**Record:** `7,85 g / 120 gr Open Tip G573` (8mm / .323 diameter)
- **DB value:** `weight_grains = 180.0`
- **Correct value:** `weight_grains = 120.0`
- **Source URL:** https://www.lapua.com/product/8-mm-cal-open-tip-bullet-g573-78-g-120-gr

The product name, URL slug, and page title all say "120 gr / 7.85g". The pipeline extracted 180gr from a specs table that appears to show data for a different bullet (the 180gr Naturalis N559). This also caused a false duplicate detection with the actual 180gr Naturalis at the same diameter.

## Queries Run

### Referential integrity
```sql
SELECT 'orphan_cartridge_bullet', COUNT(*) FROM cartridge c LEFT JOIN bullet b ON c.bullet_id = b.id WHERE b.id IS NULL;
-- Result: 0 (same pattern for all FK checks)
```

### Cartridge-bullet weight mismatch
```sql
SELECT c.name, c.bullet_weight_grains, b.weight_grains, b.name
FROM cartridge c JOIN bullet b ON c.bullet_id = b.id
WHERE ABS(c.bullet_weight_grains - b.weight_grains) > 0.5;
-- Result: 70 rows (4 severe, 56 moderate, 10 slight)
```

### Bullet diameter vs caliber diameter
```sql
SELECT c.name, cal.bullet_diameter_inches, b.bullet_diameter_inches
FROM cartridge c JOIN bullet b ON c.bullet_id = b.id JOIN caliber cal ON c.caliber_id = cal.id
WHERE ABS(b.bullet_diameter_inches - cal.bullet_diameter_inches) > 0.002;
-- Result: 0 rows
```

### Duplicate detection
```sql
SELECT m.name, b1.name, b2.name, b1.bullet_diameter_inches, b1.weight_grains
FROM bullet b1 JOIN bullet b2 ON b1.manufacturer_id = b2.manufacturer_id
  AND b1.bullet_diameter_inches = b2.bullet_diameter_inches
  AND b1.weight_grains = b2.weight_grains AND b1.id < b2.id
JOIN manufacturer m ON b1.manufacturer_id = m.id;
-- Result: 70 pairs across 8 manufacturers
```

### BC range check
```sql
SELECT b.name, m.name, b.bc_g1_published FROM bullet b JOIN manufacturer m ON b.manufacturer_id = m.id
WHERE b.bc_g1_published IS NOT NULL AND (b.bc_g1_published < 0.050 OR b.bc_g1_published > 0.900);
-- Result: 14 G1 outliers, 3 G7 outliers (all explainable by caliber)
```

### Missing BC fields (rifle bullets, excl. Cutting Edge & Nosler)
```sql
SELECT m.name, COUNT(*) FROM bullet b JOIN manufacturer m ON b.manufacturer_id = m.id
WHERE b.bc_g1_published IS NULL AND b.bc_g1_estimated IS NULL
  AND b.bc_g7_published IS NULL AND b.bc_g7_estimated IS NULL
  AND b.bullet_diameter_inches <= 0.375
  AND m.name NOT IN ('Cutting Edge Bullets', 'Nosler')
GROUP BY m.name;
-- Result: 28 bullets across 6 manufacturers
```
